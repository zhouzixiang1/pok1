5. Score against the criteria below

# Scoring Criteria (1–10)

| Score | Meaning |
|---|---|
| **9–10** | Changes directly address a confirmed weakness. Novel, high-EV improvement. Example: adds per-street opponent fold-rate tracking that adjusts continuation bet sizing |
| **7–8** | Solid changes with clear strategic rationale. Measurable positive expected value. Minor execution risk |
| **5–6** | Superficial — e.g. changing a constant by 5% with no analysis basis, minor refactors with no strategic significance. Likely won't help |
| **3–4** | Changes likely to cause regression. Wrong strategic direction. Example: increasing passive-call threshold when match analysis shows opponent exploiting passivity |
| **1–2** | Changes will clearly hurt. Catastrophic strategic errors or complete misfire on the problem |

**Score ≥ 6 → `approved: true`. Score < 6 → `approved: false` (triggers intra-generation worker retry).**

# Strategic Checklist (check each before scoring)

- [ ] **Addresses confirmed weakness**: Does the change target a pattern from recent match analysis or experience pool?
- [ ] **Opponent modeling**: Does it improve per-street tracking, bet-sizing detection, or exploitative adjustment?
- [ ] **EV basis**: Are decisions based on equity/pot-odds/fold-equity rather than arbitrary thresholds?
- [ ] **No regression on critical spots**: AA/KK/QQ still raises preflop; nut hands still value-bet river?
- [ ] **Diversity check**: Is this substantially different from the last 2 generations' approach? If same type of change (e.g. constant tuning) failed twice, it should NOT be repeated without new insight.

# Local Optima Detection

Flag `local_optima_warning: true` if:
- The change is the same TYPE as the previous 1-2 generations (e.g. constant tuning again with no new analysis)
- The experience pool has a `[POSSIBLY EXHAUSTED]` tag on a related lesson
- The change is incremental when a structural rethink is needed

## Strategic Assessment

### What v22 actually changes (vs v21):

**Worker 1's task** — Port 5 v12 features:
1. ✅ `river_thin_value_profile` — New function in `bluff_analysis.py`. Strict eligibility (top_pair/overpair only, kicker≥10, no dynamic boards, risk<0.05). Called in `get_action()` when `round_idx == 3` and `win_rate >= 0.52`. This is **genuinely new** — v21 had no thin value betting on river.
2. ✅ `river_bluff_ev` — New function in `bluff_analysis.py`. EV-based bluff recommendation using `fold_to_raise` + blocker value. Integrated into `river_blocker_bluff` condition in `get_action()`. **Genuinely new.**
3. ✅ `donk_bet_profile` — New function in `bluff_analysis.py`. OOP lead betting on wet/paired/dynamic boards with strong/nut hands or combo draws. Called in `get_action()` for donk betting path. **Genuinely new.**
4. ✅ `spr_profile` — New function in `betting.py`. Stack-to-pot ratio awareness with commitment levels. Used in `get_action()` for threshold adjustments (shallow→lower commitment thresholds by 0.02-0.04) and sizing scaling for donk bets. **Genuinely new.**
5. ✅ `turn_barrel_profile` — New function in `bluff_analysis.py`. Turn continuation barrel analysis for strong/nut hands or semi-bluffs. Integrated into the main betting decision in `get_action()`. **Genuinely new.**

**Worker 2's task** — Tune preflop thresholds to reduce fold rate:
- ❌ **Not done.** The preflop.py and preflop_strategy.py files are **identical** to v21. No threshold changes at all. The `open_threshold`, `call_threshold`, `limp_threshold`, `sb_vs_reraise` thresholds are unchanged.

### Strategic Evaluation:

**Positives:**
- The 5 ported features address real gaps identified in the experience pool: thin river value, donk betting, SPR awareness, EV-based bluffing, turn barrels.
- The `river_thin_value_profile` has **strict conditions** matching the experience pool's warning about v17's loose version being net negative.
- The `river_bluff_ev` provides a principled EV framework rather than threshold-only bluffing.
- SPR commitment adjustments are conservative (0.02-0.04 threshold shifts).
- The `turn_barrel_profile` adds a legitimate weapon for continuation betting with strong hands.

**Negatives:**
- Worker 2's task was completely ignored — preflop thresholds are unchanged.
- The donk bet is only triggered OOP with strong/nut hands on specific boards. While sound, it's conservative and may fire rarely.
- The `river_thin_value_profile` requires `len(public_cards) < 5` to return early (only 5-card rivers), and requires both `tier == "thin"` or `0.40 <= made_strength <= 0.58`, PLUS `pair_type in ("top_pair", "overpair")` with kicker≥10. The intersection of all these conditions is extremely narrow.
- The turn barrel is only for round_idx == 2 with strong/nut hands or high-quality semi-bluffs on non-wet boards. Also narrow.
- Dead code duplication creates maintenance risk and confusion.

**No regression risk**: The core decision logic is identical to v21 except for the 5 additive features. AA/KK/QQ still raises preflop. Nut hands still value-bet. The new features only add paths, they don't change existing paths.
