5. Score against the criteria below

# Scoring Criteria (1–10)

| Score | Meaning |
|---|---|
| **9–10** | Changes directly address a confirmed weakness. Novel, high-EV improvement. Example: adds per-street opponent fold-rate tracking that adjusts continuation bet sizing |
| **7–8** | Solid changes with clear strategic rationale. Measurable positive expected value. Minor execution risk |
| **5–6** | Superficial — e.g. changing a constant by 5% with no analysis basis, minor refactors with no strategic significance. Likely won't help |
| **3–4** | Changes likely to cause regression. Wrong strategic direction. Example: increasing passive-call threshold when match analysis shows opponent exploiting passivity |
| **1–2** | Changes will clearly hurt. Catastrophic strategic errors or complete misfire on the problem |

**Score ≥ 6 → `approved: true`. Score < 6 → `approved: false` (triggers intra-generation worker retry).**

# Strategic Checklist (check each before scoring)

- [ ] **Addresses confirmed weakness**: Does the change target a pattern from recent match analysis or experience pool?
- [ ] **Opponent modeling**: Does it improve per-street tracking, bet-sizing detection, or exploitative adjustment?
- [ ] **EV basis**: Are decisions based on equity/pot-odds/fold-equity rather than arbitrary thresholds?
- [ ] **No regression on critical spots**: AA/KK/QQ still raises preflop; nut hands still value-bet river?
- [ ] **Diversity check**: Is this substantially different from the last 2 generations' approach? If same type of change (e.g. constant tuning) failed twice, it should NOT be repeated without new insight.

# Local Optima Detection

Flag `local_optima_warning: true` if:

- The experience pool has a `[POSSIBLY EXHAUSTED]` tag on a related lesson
- The change is incremental when a structural rethink is needed

## Analysis Summary

### Changes Made (v25 → v26)

**Three files changed:**
1. **`state.py`** (+6 lines): Widened tier 4 preflop classification — any King, Q6o+, J7o+, 3-gappers (T7o, 96o, 85o)
2. **`betting.py`** (~80 lines changed): Major overhaul of preflop play + sizing + postflop fold logic
3. **`strategy.py`, `postflop.py`, others**: **NO CHANGES** (0 diffs)

### Specific Changes:

**A. Bet Sizing (blanket increases):**
- Nut value ratio: 0.85 → 1.00 (full pot)
- Thin/strong ratios: 0.55/0.75 → 0.75/0.92
- All preflop raise bases increased (2.5→3.0 BB, 3.5→4.5 BB, 2.8→4.0 BB, etc.)
- Min open-raise floor: 3.0 → 3.5 BB
- `thin_cap`: 0.30/0.38 → 0.40/0.48
- `low_ratio`: across-the-board increases

**B. Preflop Range (dramatically wider):**
- SB open: Now raises tier 4; tier 5 always completes ("NEVER fold from SB in HU")
- BB vs limp: Raises tier 4 (was check)
- BB vs raise: New 3-bet mechanism — top 40% hands, 50% pseudo-random frequency, 3x sizing
- BB vs raise: Calls tiers 1-4, only folds tier 5 vs >5 BB raises
- SB vs reraise: Calls tiers 1-3, tier 4 if ≤6 BB, tier 5 if ≤4 BB
- Fallback: "Never fold preflop in HU" for unknown spots

**C. Postflop Fold Logic (refined):**
- Replaced thin value protection with top-pair-or-better (≥0.45)
- Added `double_barrel` detection (prior street + current street betting)
- Redefined categories: `no_pair` (<0.18), `weak` (<0.30), added `no_strong_draw`
- Lowered fold trigger thresholds (>75% → >60% pot)

### Strategic Evaluation:

**Positives:**
- The "never fold from SB in HU" policy is correct HU strategy — it directly addresses the 57% preflop fold rate
- The 3-betting mechanism with pseudo-random coinflip is a genuine structural improvement — it wasn't in v24 or v25
- Double-barrel detection (prior street + current street) is a real improvement in opponent aggression tracking
- Wider BB defense is sound HU theory

**Major Concerns:**
1. **Third consecutive generation of "wider preflop + bigger sizing"**: v24 did this ("Increased bet sizing ratios... Further widened preflop ranges"), v25 did this ("Increased bet sizing multipliers from 0.3-0.5x to 0.6-0.8x pot... 5-tier preflop classification"), and now v26 does the same thing again with even larger numbers. **This is a repeating pattern without new insight.**

2. **v25 already regressed from its parent**: v25 branched from v9 (r=1583.0) but landed at r=1559.7 — a 24-point drop. v25's approach (which v26 extends) hasn't been validated.

3. **Zero postflop engine changes**: The bot now plays wider ranges and builds bigger pots, but `strategy.py` (993 lines) and `postflop.py` (999 lines) are completely untouched. The postflop decision engine that must handle these wider ranges in bigger pots hasn't been improved at all.

4. **Blanket sizing increase is not targeted**: Every single sizing constant went up. The experience pool specifically warns "Wholesale copy fails. Incremental targeted port wins."

5. **No match-data-driven justification**: v25 has 0 head-to-head entries in the data — there's no evidence about which opponents v25 loses to, which streets leak chips, or what patterns to target.
