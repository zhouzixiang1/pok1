### Issues Found

#### 🔴 CRITICAL BUG: Missing `TOTAL_HANDS` Import in `betting.py`
The `choose_anti_lock_pressure_action` function was **moved** from `strategy.py` (v28) to `betting.py` (v29), but `TOTAL_HANDS` was not added to `betting.py`'s imports. Line 554 of `betting.py`:
```python
hands_left = remaining_hands if remaining_hands is not None else TOTAL_HANDS
```
`betting.py` only imports `BIG_BLIND` from constants — **`TOTAL_HANDS` is not in scope**. This will raise `NameError` at runtime when `remaining_hands=None` during anti-lock pressure situations (late-game tournament spots). This is a **crash bug** that would cause the bot to time out and lose.

#### 🟡 STRATEGIC REGRESSION: River 3-Branch Preempts Nut Overbet
The new `river_3branch_decision` (line 357 of strategy.py) executes **before** the existing nut overbet logic (line 377). For nut-tier hands, `classify_river_hand` returns "strong", and the strong branch bets 60-75% pot. The overbet code (1.5-2.2x pot with nuts) becomes **dead code** — it can never be reached on the river. This reduces maximum value extraction from the strongest hands on dry boards.
