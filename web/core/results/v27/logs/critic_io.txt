# Scoring Criteria (1–10)

| Score | Meaning |
|---|---|
| **9–10** | Changes directly address a confirmed weakness. Novel, high-EV improvement. Example: adds per-street opponent fold-rate tracking that adjusts continuation bet sizing |
| **7–8** | Solid changes with clear strategic rationale. Measurable positive expected value. Minor execution risk |
| **5–6** | Superficial — e.g. changing a constant by 5% with no analysis basis, minor refactors with no strategic significance. Likely won't help |
| **3–4** | Changes likely to cause regression. Wrong strategic direction. Example: increasing passive-call threshold when match analysis shows opponent exploiting passivity |
| **1–2** | Changes will clearly hurt. Catastrophic strategic errors or complete misfire on the problem |

**Score ≥ 6 → `approved: true`. Score < 6 → `approved: false` (triggers intra-generation worker retry).**

# Strategic Checklist (check each before scoring)

- [ ] **Addresses confirmed weakness**: Does the change target a pattern from recent match analysis or experience pool?
- [ ] **Opponent modeling**: Does it improve per-street tracking, bet-sizing detection, or exploitative adjustment?
- [ ] **EV basis**: Are decisions based on equity/pot-odds/fold-equity rather than arbitrary thresholds?
- [ ] **No regression on critical spots**: AA/KK/QQ still raises preflop; nut hands still value-bet river?
- [ ] **Diversity check**: Is this substantially different from the last 2 generations' approach? If same type of change (e.g. constant tuning) failed twice, it should NOT be repeated without new insight.

## Poker Strategy Critic Evaluation

### What I Can Assess (from Master's Plan)

The Master assigned **3 changes** to the Algorithmic Logic Architect:

1. **Add draw equity protection to fold gate** — Prevents folding hands that have sufficient draw equity (flush/straight draws) to continue profitably
2. **Restore anti-bot4 detection** — Reintroduces opponent-specific exploitation logic targeting bot4
3. **Increase bet sizing constants** — Tuning bet sizes upward

### Strategic Evaluation

**Change 1: Draw equity protection to fold gate — STRATEGICALLY SOUND ✅**
- This addresses a well-known poker bot leak: folding draws too aggressively when pot odds justify continuing
- If the fold gate previously used only hand strength (made hand equity) without considering draw equity, this is a genuine structural improvement
- Correctly implementing this requires comparing draw equity (outs × street-appropriate factor) against pot odds being laid — this is fundamentally +EV
- **Risk**: If draw equity is overestimated (e.g., not discounting dirty outs), it could lead to calling too light

**Change 2: Restore anti-bot4 detection — CONTEXT-DEPENDENT ⚠️**
- Opponent-specific exploitation is strategically valid IF bot4 is a frequent matchup
- The word "restore" suggests this was previously present and was removed — this implies a regression was identified
- **Concern**: Hardcoding opponent-specific adjustments can create brittleness. If this is pattern-based detection (e.g., "if opponent raises exactly X% of the time, adjust Y way") it's fragile
- **Better approach**: Would be a general opponent modeling system that learns tendencies, but restoring a known-good heuristic is acceptable as a short-term fix

**Change 3: Increase bet sizing constants — WEAKEST CHANGE ⚠️**
- Constant tuning without match analysis basis is the most common type of low-value change in poker bot evolution
- Without seeing the analysis showing that the current sizing is too small (e.g., opponents calling too profitably against small bets), this is arbitrary
- The CLAUDE.md explicitly warns about this pattern — changing constants by small amounts with no analytical justification

### Diversity / Local Optima Check

- The plan includes **3 different types** of changes (structural logic, opponent modeling, constant tuning) — this is good diversity within a single generation
- However, **constant tuning (Change 3)** is a very common incremental change. If previous generations also focused on sizing adjustments, this is treading well-worn ground
- The **branch from bot-v7** (20 versions back) is interesting — it suggests the Master identified that recent versions degraded and went back to a known-good base. This is a reasonable strategic reset

### Critical Concern — Cannot Verify Implementation

**I was unable to read the actual code changes** due to tool limitations. This means I cannot verify:
- Whether the draw equity calculation is correct (outs counting, equity estimation)
- Whether anti-bot4 detection is truly restored or partially implemented
- The magnitude of bet sizing changes (5%? 50%?)
- Whether critical hands (AA/KK/QQ preflop, nut hands river) are still handled correctly
- Whether any regressions were introduced
