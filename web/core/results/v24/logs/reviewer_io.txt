## Code Review: claude_v24 vs claude_v23

### Scope of Changes
Only **2 files** actually changed between v23 and v24: `strategy.py` (+7 lines) and `strategy_helpers.py` (+43 lines / ~15 modified lines). The git diff showed 8188 insertions because it compared against the tag, not the directory — the 20 other files are identical.

### Task Verification

**Task 1 — `should_fold_postflop()` for 0% → 15-25% fold rate ✅**
- New function (lines 580-618) with sound logic: protects nut/strong hands and strong draws, folds air vs pressure on later streets, folds weak pairs vs big bets.
- Integrated in `strategy.py` line 373-379 with proper safety gates (`anti_lock_call_continue` and `strong_made_continue`).
- Only triggers postflop (`round_idx > 0`), so AA/KK/QQ/AKs preflop is unaffected.

**Task 2 — Bet sizing increase from 0.4x → 0.65-0.75x ✅**
- `choose_raise()` base ratios raised: flop 0.60→0.65, turn 0.70→0.75, preflop 0.55/0.75→0.60/0.80. River unchanged at 0.85.
- Low ratio floor: 0.40→0.50. Postflop sizing now squarely in the 0.65-0.85 range.

**Task 3 — Preflop fold rate reduction ✅**
- SB open: 0.43→0.38, limp: 0.27→0.22, BB vs limp iso: 0.50→0.44, BB vs raise call: 0.34→0.28, SB vs reraise premium: 0.68→0.62, continue: 0.52→0.45.

### Correctness Checks
- ✅ All 20 `.py` files compile cleanly
- ✅ All module imports resolve
- ✅ Bot runs in battle engine (1-1 draw vs v23 in 2-game smoke test)
- ✅ No file exceeds 1000 lines (largest: strategy.py at 751)
- ✅ JSON protocol correct: `json.loads(input())` → `json.dumps({"response": int(action)})`
- ✅ No stray `print()` or `input()` calls
- ✅ Only stdlib imports used

### Minor Concerns
1. **`pot_odds` parameter unused**: Passed to `should_fold_postflop` but the function uses `bet_ratio = to_call/pot` instead. Harmless but worth cleaning up.
2. **Aggressive preflop widening**: The combined effect of all threshold reductions is significant — the bot will play substantially more hands preflop, which increases variance and could create tougher postflop spots.
3. **`main_backup.py` is 2941 lines** but is pre-existing (unchanged from v23) and appears to be a dead backup file — not imported anywhere.
