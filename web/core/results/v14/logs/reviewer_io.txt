## Review Summary

**Crossover Analysis: v2 → v14**

The workers performed a crossover combining v2's well-calibrated core with v14's advanced analysis functions. Here are the key changes:

### Positive Changes
1. **`min_raise_action` fix in `state.py` + `strategy.py` + `main.py`** — Correctly computes minimum legal raise as `round_raise - my_round_bet`. Uses it throughout instead of raw `round_raise`. This fixes a potential bug where the bot could return illegally small raises.

2. **`must_continue_vs_raise()` in `strategy.py`** — New safety net preventing folds with nut/strong hands against raises. Nut tier always continues; strong hands continue with reasonable pot odds and low nutted risk.

3. **`check_probe_resistance_margin()` moved from `postflop.py` to `strategy.py`** — Better module organization; function is only used by strategy.py.

4. **Safe exploitation threshold mixing** — GTO baselines (`gto_strong`, `gto_medium`) are computed and blended with exploit thresholds via `exploit_lambda`. This is more robust than pure exploitation.

5. **Opponent model refactored in `opponent.py`** — Inlined `_track_hand_stats` into the main loop. Functionally equivalent, avoids per-hand dict creation overhead.

6. **`thin_static_showdown_control`** — New check: thin-value hands on static boards with no draws prefer checking to showdown. Reduces getting raised off marginal equity.

7. **`sanitize_action` bug fix in `main.py`** — Changed `return 0 if to_call == 0 else -1` to `return 0` for sub-minimum raises. If action is between `to_call` and `min_raise`, calling is correct (not folding).

8. **3bet/4bet simplification** — Cleaner preflop raise-or-fold logic with proper all-in conversion when sizing exceeds half stack.

### Concerns
- **`strategy.py` is 1245 lines**, exceeding the 1000-line constraint. However, this was inherited from v2 (1216 lines) — the workers added only 29 lines. This is a pre-existing violation.
- **`main_backup.py` at 2941 lines** — Dead code, not imported anywhere. Should be cleaned up but doesn't affect functionality.
- **Removed adaptive 3bet sizing** — v2 had `three_bet_mult = 3.0 + clamp(fold_to_raise - 0.44, -0.5, 0.5)` with a fallback. v14 uses fixed 0.75 pot ratio. Simpler but less exploitative.
- **Removed `call_threshold -= confidence * max(0.0, fold_to_raise - 0.50) * 0.04`** — Small call threshold reduction for tight raisers is gone.
