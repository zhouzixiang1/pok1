# Scoring Criteria (1–10)

| Score | Meaning |
|---|---|
| **9–10** | Changes directly address a confirmed weakness. Novel, high-EV improvement. Example: adds per-street opponent fold-rate tracking that adjusts continuation bet sizing |
| **7–8** | Solid changes with clear strategic rationale. Measurable positive expected value. Minor execution risk |
| **5–6** | Superficial — e.g. changing a constant by 5% with no analysis basis, minor refactors with no strategic significance. Likely won't help |
| **3–4** | Changes likely to cause regression. Wrong strategic direction. Example: increasing passive-call threshold when match analysis shows opponent exploiting passivity |
| **1–2** | Changes will clearly hurt. Catastrophic strategic errors or complete misfire on the problem |

**Score ≥ 6 → `approved: true`. Score < 6 → `approved: false` (triggers intra-generation worker retry).**

If the assessment is only "looks reasonable" and does not cite a validation path
from H2H weakness, experience pool, and actual diff, the maximum score is 6.
If the change repeats a prior failed or `[POSSIBLY EXHAUSTED]` pattern without new evidence,
the maximum score is 5 and `local_optima_warning` must be true.

# Strategic Checklist (check each before scoring)

- [ ] **Addresses confirmed weakness**: Does the change target a pattern from recent match analysis or experience pool?
- [ ] **Opponent modeling**: Does it improve per-street tracking, bet-sizing detection, or exploitative adjustment?
- [ ] **EV basis**: Are decisions based on equity/pot-odds/fold-equity rather than arbitrary thresholds?
- [ ] **No regression on critical spots**: AA/KK/QQ still raises preflop; nut hands still value-bet river?
- [ ] **Diversity check**: Is this substantially different from the last 2 generations' approach? If same type of change (e.g. constant tuning) failed twice, it should NOT be repeated without new insight.

- exact_wr > 0.85 + facing bet → value raise
- exact_wr < 0.35 + facing bet → fold if below pot odds - 0.05
- exact_wr < 0.20 + no bet + blockers + non-passive opponent → enable river blocker bluff

- Apply `style_deltas` to `strong`/`medium` decision thresholds
- Subtract `bluff_freq_bonus` from all 3 bluff thresholds (river_bluff, probe_fold, semi_bluff)

## Summary of Actual Changes (v8 → v30)

The diff between v8 and v30 reveals **two focused changes**:

### 1. `opponent.py`: Added `classify_opponent_style()` (32 new lines)
A new function that classifies opponents into 4 archetypes — **nit** (low vpip, high fold), **maniac** (high vpip/pfr/aggr), **station** (high vpip, low pfr, low fold), **fold-heavy** (high fold_to_raise) — and returns threshold deltas.

### 2. `strategy.py`: Integration of style_deltas + river exact equity (~30 lines changed)
- Computes `style_deltas = classify_opponent_style(opponent_model)` (line 380)
- Propagates `style_deltas['strong_delta']` and `style_deltas['medium_delta']` to decision thresholds (lines 504-505)
- Propagates `style_deltas['bluff_freq_bonus']` to all three bluff thresholds (river, probe, semi-bluff) (lines 905-907)
- Adds river exact equity enumeration when 5 public cards available, with precise raise/fold/blocker-bluff branches (lines 410-433)

### Key Observations

**Strengths:**
- Directly implements experience pool lessons: "classify_opponent_style returns threshold deltas — +2-3 pts adaptation" and "style_deltas MUST propagate to ALL bluff thresholds AND strong/medium decision thresholds"
- River exact equity matches experience pool: "River exact equity override: when 5 public cards, use exact enumeration (0 sims) for precise raise/fold decisions"
- v8's weakest H2H matchups (v15: 36%, v9: 45%, v4: 45.7%) suggest poor opponent adaptation — exactly what this addresses
- The experience pool explicitly identifies missing classify_opponent_style as a v17 failure cause

**Weaknesses:**
- Two of the five computed deltas (`call_aggression_bonus`, `fold_vs_passive_bonus`) are dead code — never consumed in strategy.py
- Deltas are conservative (-0.02 to +0.03), may be too small for measurable win-rate impact alone
- Doesn't specifically target v8's worst matchup patterns (v15 at 36%)

**Diversity:** Different change type from recent generations (v28: board texture, v29: preflop tiers + river branches). This is opponent-modeling focused — structurally novel.
